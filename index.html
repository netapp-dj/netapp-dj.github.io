🗓 Proposed Five-Phase Flow
P0
Before Day 1
Pre-Arrival Brief
New hire arrives with mental model already forming — not starting cold
▶
Buddy / Manager sends ahead
  • Send a single-page "What to expect on Day 1" brief (not the full Confluence page)
  • Link to the Grid Primer — it works outside VPN and is a 45-minute read
  • Share David Leung's intro video — the one product overview everyone should see
  • Pre-submit access requests: Bitbucket, Mantis, Jira — these take days
  • Reserve a block of ICT/RTP IPs so the new hire is unblocked on Module 3
✦ AI Opportunity
Generate a personalised "skills mapping" brief using the new hire's resume and Claude. e.g. "You know Kubernetes distributed state — here's how StorageGRID's Cassandra metadata cluster is conceptually similar." A 1-page custom orientation document rather than a generic welcome letter.

💡 Why reference grid first, own grid later
New hires can click around ILM, Tenants, and the Dashboard on a pre-built reference grid on Day 3 — building intuition before Module 3 asks them to install their own. Module 3 then feels like confirming understanding rather than navigating in the dark.
Terminology — change the approach
  • Keep the Initialisms and Key Terms pages — but as a contextual reference, linked from each module, not assigned as front-loaded required reading
  • Add a "glossary" sidebar link to each module page
✦ AI Opportunity
AI-generated self-check quizzes after the primer read — 5 questions like "What is the minimum number of storage nodes for data redundancy?" with explanations. No grade, just reflection. Keeps learning active rather than passive. Trivial to generate with the Claude API given the primer text.
✦ AI Opportunity
Auto-summarise internal Zoom recordings (Whisper transcription → Claude summary) so new hires can scan a 2-paragraph summary before committing 45 minutes to a video. Dead links could be flagged automatically too.
P4
Week 1, Days 4–5
Git Exercise + First Contribution
Make your first real contribution. Understand the review culture.
▶

The existing Module 2 is well-structured and has a real deliverable — keep the core but add these improvements:

  • Add one paragraph contextualising why MrClean exists before the exercise: "This is the automation repo that verifies StorageGRID behaves correctly for customers — every test you write prevents a future bug."
  • Move the "good PR description" guidelines from §6.1 to the top — it's easy to miss when buried

✦ AI Opportunity (already in place)
Claude Code PR review on MrClean PRs — validates test structure, flags common mistakes, suggests better assertions. This is already partially available; formalise it in the onboarding so new hires know to use it.

✦ AI Opportunity: Grid Doctor
After grid installation, new hire pastes their admin IP into a script. An AI checks: grid health, storage node count, ILM policy validity, tenant existence, storage watermarks. Returns a structured "your grid looks healthy / here are 2 warnings" report. Replaces the vague "navigate and check out the features" instruction.
✦ AI Opportunity: .ini/.json Editor Guard
Before running the deploy scripts, an AI reads the new hire's .ini and .json files and flags: IP addresses not in their reserved range, duplicate IPs, missing required fields, wrong datastore names. Prevents the "you owe 30 people coffee" class of mistakes.

🔧 Non-AI Structural Improvements
1.
Urgency tiers on Day 1. Separate "Must do before lunch" / "Before end of week" / "Whenever". Stop competing printers with MFA.
2.
Buddy prep guide. A parallel checklist for the buddy — what to prepare before Day 1, what to cover in Week 1 1:1s, what questions to expect. The buddy system is the right foundation; give buddies the structure to be consistent.
3.
Site-specific callouts. Clearly flag VTC / RTP / NB steps. Don't bury them in general prose where someone on a different continent follows the wrong instructions.
4.
Dead content audit. Module 3 still references SG 11.2, old wikid.netapp.com and confluence.eng.netapp.com URLs, RTP vSphere creds in plaintext. One afternoon of link-checking would fix most of it.
5.
Video curation. Remove dead links. Add durations. Add a one-sentence "what you'll learn" for each. A table of 4 curated videos beats a wall of 12 unverified links.
6.
Feedback loop. 30-day and 60-day anonymous survey for new hires. "What confused you? What was missing? What was great?" Iterate the docs every cohort, not every two years.
7.
Rookie Cookies goes on Day 1 page. Currently buried in a sidebar note in QA Module 2 — only QA new hires find it. Put it prominently on the main page so everyone knows about it regardless of role.
8.
GenAI tools page on Day 1. Move the GenAI Tools Confluence link from a table cell on the main page to a named step: "Set up your AI tools on Day 1 — here's what the team uses and how."
